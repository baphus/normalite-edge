## 3. What's included at this price

- Full source code and deployment configuration (already live on Vercel/Render/Supabase)
- All features listed in the Project Summary, as currently deployed
- [DEPLOYMENT.md](DEPLOYMENT.md) and [GETTING_STARTED_GUIDE_v1.0.0.md](GETTING_STARTED_GUIDE_v1.0.0.md) documentation

## 4. What's not included (separate line items)

- **Hosting/infrastructure costs** (Render, Supabase, Vercel, Cloudinary) — these are pass-through platform fees, billed separately and scaling with usage; see prior cost estimates for tiers from pilot to full rollout.
- **Ongoing maintenance/support** beyond the delivered scope — recommend a small recurring retainer if continued bug fixes and feature work are expected.
- **Email notification integration** (currently not implemented — see Known Limitations in the Project Summary) — would be a separate scoped addition if the university wants to fund it.

## 5. Payment note

Since payment will need to go through the university's disbursement process, confirm with their finance/BAC office whether this should be structured as a **Job Order / Contract of Service** (typical for individual contractors without business registration) rather than a commercial invoice — this determines what paperwork (TIN, signed contract/MOA) is needed before payment can be processed.
